# PR Review

## 워크플로

1. `gh pr diff {PR번호}` 로 diff 가져오기
2. 다음 기준으로 분석:
   - **완성도**: 요구사항 충족 여부
   - **코드 품질**: 컨벤션, 가독성, 타입 안전성
   - **버그/이슈**: 엣지케이스, 잠재적 오류
   - **개선 제안**: 더 나은 방법이 있으면 제안
3. 결과를 Discord #코딩 채널에 전송

## 리포트 형식

```
## 🔍 PR #{번호} 리뷰 — {제목}

📋 **완성도**: {평가}

🐛 **발견한 이슈**:
- {이슈1}
- {이슈2} (없으면 "없음")

💡 **개선 제안**:
- {제안1}
- {제안2} (없으면 "없음")

🏁 **최종 평가**: ✅ 머지 가능 / ⚠️ 수정 후 머지 / ❌ 재작성 필요
> {한 줄 코멘트}
```

## 주의사항

- `source ~/.config/envman/PATH.env` 후 gh 명령 실행
- diff가 너무 크면 주요 파일만 집중 분석
- docs 전용 PR은 내용 정확성 + 가독성 위주로 리뷰
- 결과는 **반드시 Discord #코딩 채널(1481637991637192796)로만** 전송
- 명령채널에는 "리뷰 시작합니다" 1줄만
