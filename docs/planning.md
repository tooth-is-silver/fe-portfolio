# Planning Document — Frontend Portfolio

> 작성일: 2026-03-13 | 상태: 초안 (v1)

---

## 1. 목표

프론트엔드 개발자로서의 역량과 프로젝트를 보여주는 **단일 페이지 포트폴리오 사이트**.  
채용 담당자/클라이언트가 5분 안에 핵심을 파악할 수 있도록 심플하고 임팩트 있게.

---

## 2. 타겟

- IT 기업 채용 담당자
- 협업 개발자 / 클라이언트
- 기술 스택 검증이 필요한 상대방

---

## 3. 기술 스택

- React 19 + TypeScript + Vite
- CSS (순수 CSS 또는 경량 라이브러리 — 추후 결정)
- 배포: 미정

## 6. 디자인 방향

| 항목 | 방향 |
|------|------|
| 색상 | 라이트 테마 기본, 다크 모드 (v2) |
| 폰트 | Pretendard (한국어) + 시스템 폰트 |
| 분위기 | 깔끔하고 전문적, 군더더기 없이 |
| 애니메이션 | 스크롤 페이드인 정도만 (v1), 더 풍부하게 (v2) |
| 반응형 | 모바일 퍼스트 |

## 8. 개발 단계

| Phase | 내용 |
|-------|------|
| **v1 (현재)** | 기본 레이아웃, 더미 데이터, 반응형 |

| **v3** | 상세 프로젝트 모달, 연락 폼 |

---

## 9. PR/이슈 관리 규칙

- 작업 단위: **PR 기반** (feature 브랜치 → main)
- QA 버그: **GitHub Issues**로 등록
- 이슈 해결: PR/커밋 메시지에 `Closes #이슈번호` 연결
- 브랜치 네이밍: `feature/섹션명`, `fix/이슈번호-설명`
