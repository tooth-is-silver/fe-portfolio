# 002 — 테스트 범위: 순수 함수 유틸만 TDD

**날짜**: 2026-03-14  
**상태**: 확정

## 고민

포트폴리오에 테스트 코드가 필요한가?  
컴포넌트까지 테스트하면 오버킬 아닌가?

## 결정

순수 함수(유틸) 한정으로만 TDD 적용. 컴포넌트는 테스트 안 쓴다.

## 이유

**테스트 가치가 높은 것:**
- 순수 함수 (getActiveSection, truncateText, groupBy)
- 입력 → 출력만 검증, 브라우저 없이 실행 가능
- 엣지케이스 버그가 나기 쉬운 로직 (`scrollY 음수`, `maxLength 0` 등)

**테스트 가치가 낮은 것:**
- 포트폴리오 컴포넌트 렌더링 테스트
- 규모 대비 설정/유지비가 크고 변경 시마다 수정 필요

## 결과

- 유틸 3개 (getActiveSection / truncateText / groupBy) 총 16개 테스트 작성
- CI에서 자동 실행, 실패 시 머지 블록
